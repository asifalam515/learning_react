লার্ন উইথ সুমিতের রিয়্যাক্টিভ এক্সিলারেটর কোর্সের চতূর্থ ব্যাচের প্রথম এসাইনমেন্টে আপনাকে স্বাগতম। এই মডিউলে শেখানো বিষয়গুলোর উপর ভিত্তি করে আপনাদের জন্য একটি প্র্যাকটিক্যাল এসাইনমেন্ট তৈরি করা হয়েছে। আমরা ইতিমধ্যে HTML এবং Tailwind CSS ব্যবহার করে একটি প্রস্তুত টেমপ্লেট দিয়েছি। আপনার কাজ হবে সেই টেমপ্লেট ব্যবহার করে নির্ধারিত এসাইনমেন্ট সম্পন্ন করা।

এই এসাইনমেন্টের মূল উদ্দেশ্য হলো — আপনি নিজে থেকে একটি React প্রজেক্ট তৈরি করতে পারেন কিনা, TailwindCSS সঠিকভাবে সেটআপ করতে পারেন কিনা, এবং একটি HTML টেমপ্লেটকে React কম্পোনেন্টে **Pixel Perfect** রূপে রূপান্তর করতে সক্ষম কিনা তা যাচাই করা।

**দ্রষ্টব্য:** এই এসাইনমেন্টে কোনোভাবেই নিজের মতো করে ডিজাইন বা টেমপ্লেট পরিবর্তন করা যাবে না। আপনাকে শুধুমাত্র এই মডিউলে শেখানো বিষয়গুলো ব্যবহার করেই Vite এবং React দিয়ে প্রজেক্টটি সম্পন্ন করতে হবে। অন্যথায় আপনার মার্কে প্রভাব পড়তে পারে।

## এসাইনমেন্টে আপনাকে যা যা করতে হবেঃ

✓ প্রথমেই একটি ফোল্ডারে গিয়ে **npm create vite@latest** কমান্ড ব্যবহার করে একটি নতুন React প্রজেক্ট তৈরি করুন ।

✓ আপনার তৈরি করা নতুন React প্রজেক্ট এ [Tailwindcss](https://tailwindcss.com/docs/installation/using-vite) ইন্সটল করুন। তবে মনে রাখবেন আপনি tailwind এর ভার্শন ৪ ব্যবহার করবেন। ভার্শন ৪ ইন্সটল করার জন্যে এই [ডকুমেন্টেশন লিংক](https://tailwindcss.com/docs/installation/using-vite) এর স্টেপ 2 থেকে ফলো করতে পারেন ।

✓ যদি আপনার TailwindCSS সঠিকভাবে ইন্সটল ও কনফিগার করা থাকে এবং React প্রজেক্টে সঠিকভাবে কাজ করে, তাহলে আমাদের দেওয়া **index.html** ফাইলের **Styles, Fonts, Meta Information সহ সম্পূর্ণ টেমপ্লেটটি React কম্পোনেন্টে রূপান্তর করতে হবে।**

✓ কম্পোনেন্ট তৈরির ক্ষেত্রে, প্রতিটি সেকশন যেমন **Hero**, **Testimonials**, **Features**, **Pricing** ইত্যাদি অংশগুলোকে আলাদা আলাদা কম্পোনেন্টে ভাগ করে কাজ করতে হবে। অর্থাৎ, **Hero.jsx** ফাইলে শুধুমাত্র Hero সেকশনের কোড থাকবে। তবে শুধু এই কয়েকটি নয় — আপনাকে পুরো প্রজেক্ট বিশ্লেষণ করে যেখানে প্রয়োজন, সেখানে ছোট ছোট **Re-usable কম্পোনেন্ট** তৈরি করতে হবে, যেন কোড আরও সংগঠিত ও পুনঃব্যবহারযোগ্য হয়।

✓ পুরো প্রজেক্টে আপনাকে **Re-usable কম্পোনেন্ট** তৈরি করতে হবে। উদাহরণ হিসেবে, **Services** সেকশনে থাকা একাধিক কার্ড — যেমন _Smart. Simple. Brilliant._, _Your work, in sync_, _Effortless integration_ — এদের ডিজাইন একই রকম। তাই এই ডেটাগুলোকে একটি **Array of Objects** আকারে রেখে, একটি মাত্র কম্পোনেন্টকে **লুপের মাধ্যমে রেন্ডার** করতে হবে এবং প্রয়োজনীয় ডেটা **props** হিসেবে পাঠাতে হবে।

একইভাবে, **Features** সেকশনের _Plan your schedules_, _Analytics & insights_, _Collaborate seamlessly_ অংশগুলোর ক্ষেত্রেও একই প্যাটার্ন অনুসরণ করতে হবে — একটি কম্পোনেন্ট ব্যবহার করে **ডেটা লুপ করে Re-render** করতে হবে।

✓ আমাদের দেওয়া **index.html** ফাইলে একাধিক svg ট্যাগ রয়েছে। প্রতিটি svg আইকনের জন্য একটি **আলাদা কম্পোনেন্ট** তৈরি করতে হবে। যেমন, কোনো জায়গায় যদি svg কোড ব্যবহার করা হয়ে থাকে, তাহলে সেই অংশটি কপি করে একটি নতুন কম্পোনেন্টে রাখতে হবে এবং সেই SVG কোডটি **return** করতে হবে। এরপর React প্রজেক্টে সরাসরি svg tag না লিখে, সেই কম্পোনেন্টটি ব্যবহার করতে হবে। উদাহরণস্বরূপ: svg tag এর পরিবর্তে <IconComponent /> লিখতে হবে।

**এসাইনমেন্টে আপনাকে মাত্র দুইটা জিনিস সাবমিট করতে হবে:**

1. **GitHub private repository link:** অবশ্যই সঠিক গিটহাব রিপোজিটরি লিংক দিতে হবে। ভুলে অন্য কোনো লিংক দিলে আপনি এসাইনমেন্টের মার্ক পাবেন না। তাই সাবমিট করার আগে নিউ ট্যাবে লিংক ওপেন করে চেক করে নিবেন সঠিক লিংক জমা দিচ্ছেন কিনা। আরেকটি বিষয় খুবই গুরুত্বপূর্ণ। আপনার প্রজেক্টে কোন এনভায়রনমেন্ট ফাইল যেমন .env অথবা .env.local থাকলে সেগুলোতে থাকা ইনফর্মেশন যদি আপনার প্রজেক্টটি কাজ করার জন্য প্রয়োজনীয় হয়, তাহলে সেগুলো অবশ্যই আপনার .gitignore ফাইল থেকে সরিয়ে নিবেন যেন আমরা সেটি দেখতে পাই। কারণ এসাইনমেন্ট রিভিউ করার সময় আমরা আপনার প্রজেক্টটি রান করে দেখবো। তাই রান করার জন্য প্রয়োজনীয় সকল ফাইল আমরা যেন দেখতে পাই, সেটা আপনাকে নিশ্চিত হতে হবে।
2. **Live site link:** নেটলিফাইতে বা ভার্সেলে সাইট হোস্ট করে সাইটের লাইভ লিংক দিতে হবে। তবে এর বাইরে অন্য কোন হোস্টিং প্রভাইডারেও আপনি চাইলে হোস্ট করতে পারে। তবে সেক্ষেত্রে আপনার হোস্টিং রিলেটেড সমস্যা নিজেকে সল্ভ করতে হবে এবং সব কিছু ঠিক মতো কাজ করতে হবে। ভুলে অন্য কোনো লিংক দিলে আপনি এসাইনমেন্টের মার্ক পাবেন না। তাই সাবমিট করার আগে নিউ ট্যাবে লিংক ওপেন করে চেক করে নিবেন সঠিক লিংক জমা দিচ্ছেন কিনা। ভার্সেলে কি ভাবে হোস্ট করতে হয় তা আপনি না জানলে [এইখানে ক্লিক](https://learnwithsumit.com/rnext/courses/rnext/how-to-deploy-your-project-to-vercel-free) করে দেখে নিতে পারেন।
